# Документация AgroLead Assistant v6

Этот пакет документов описывает текущий легаси-проект целиком: от бизнес-логики чата до схемы данных, деплоя и плана стабилизации.

## Как читать документацию

- Если ты новый разработчик: начни с `docs/08-onboarding-junior.md`.
- Если нужен полный технический контекст: читай по порядку `01 -> 08`.
- Если задача по продакшену/поднятию стенда: сразу в `docs/06-deploy-and-ops.md`.

## Карта разделов

1. `docs/01-overview.md`

2. `docs/02-architecture.md`

3. `docs/03-dialogue-and-prompts.md`

4. `docs/04-database.md`

5. `docs/05-api.md`
   - Контракты API, auth-модель, форматы запросов/ответов.
6. `docs/06-deploy-and-ops.md`

   - Пошаговый маршрут онбординга для джуна.

## Важные факты о текущем состоянии

- Backend сконцентрирован в `backend/app/main.py` (единый файл API + оркестрации).
- Миграций нет: схема создается через `SQLModel.metadata.create_all()`.
- Текущая БД в продовом сценарии — PostgreSQL (через `docker-compose.yml`).
- LLM используется как генератор формулировок, а не как источник истины.
- Много бизнес-правил реализовано регэкспами и детерминированной логикой.

## Термины

- `lead` — заявка в CRM-контуре (`crm_lead`).
- `session` — чат-сессия (`chat_session`) с привязкой к lead.
- `fact` — извлеченный факт из сообщения (`chat_extracted_fact`).
- `missing field` — обязательное поле, которое еще не собрано.
- `guardrails` — правила безопасного/делового поведения диалога.
- `RAG` — выборка релевантных статей из `knowledge_article`.
