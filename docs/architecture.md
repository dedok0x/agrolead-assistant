# Архитектура (MVP)

## Цель

Сократить рутинные предпродажные коммуникации и ускорить первичный ответ клиенту.

## Подсистемы

- Канал коммуникации (мессенджер / webhook).
- AI-модуль (интерпретация запроса, формирование ответа).
- Core API (бизнес-логика и маршрутизация сценариев).
- Хранилище данных (клиенты, товары, заявки, диалоги).
- Админ-интерфейс (просмотр новых обращений и заявок).

## Минимальные сущности

- Клиент: id, контакты, источник, статус.
- Товар: наименование, класс, цена, остаток.
- Заявка: клиент, товар, объём, комментарий, статус.
- Диалог: канал, таймстемп, реплики, итог обработки.

## Масштабирование

1. Добавление PostgreSQL и Redis.
2. Очередь сообщений при росте интеграций.
3. Разделение на сервисы: ai-service, lead-service, admin-service.
