# MVP-сценарии

## Сценарий A — Запрос цены/наличия

1. Клиент отправляет запрос по товару.
2. Система определяет товар и ищет данные.
3. Клиент получает ответ по цене и доступности.

## Сценарий B — Уточнение параметров

1. Система задаёт уточняющие вопросы (объём, класс, срок).
2. Формирует квалифицированный интерес.

## Сценарий C — Создание заявки

1. На основе подтверждённого интереса создаётся заявка.
2. Менеджер получает заявку в админ-интерфейсе.

## Сценарий D — Просмотр обращений менеджером

1. Менеджер открывает список новых заявок.
2. Просматривает контекст диалога.
3. Меняет статус обращения.

---

## Обновлённая стратегия sales-диалога (human-like)

### 1) Тон и стиль

- Короткий, живой, доброжелательный тон без канцелярита.
- Не спорить с клиентом, не повторять один и тот же вопрос.
- На каждом шаге максимум **один** следующий уточняющий вопрос.

### 2) Состояния диалога

1. `new` — первое касание, нужно вовлечь.
2. `in_progress` — собираем параметры сделки.
3. `qualified` — есть товар/класс/объём/регион + контакт.
4. `closed` — лид передан или закрыт по причине.

### 3) Порядок квалификации лида

1. Культура (`пшеница/ячмень/кукуруза`)
2. Класс/качество
3. Объём (тонны)
4. Регион поставки
5. Срок поставки
6. Контакт (телефон или email)

### 4) Анти-аномалии

- Если пользователь вводит шум (`123`) — мягко возвращать к предмету, не «ронять» диалог.
- Если пользователь передумал по продукту/классу — сбрасывать старые параметры и продолжать без конфликта.
- Вопросы типа «на какой модели работаешь» — корректный ответ + возврат к задаче клиента.

### 5) Хранение заявок

- В `Lead` сохраняются:
  - контактные данные,
  - параметры сделки,
  - `status` этапа,
  - комментарий менеджера.
- Рекомендуется дополнительно фиксировать причину потери в `comment` для аналитики.
