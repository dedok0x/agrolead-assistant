# 01. Обзор системы

## 1) Назначение проекта

AgroLead Assistant v6 — это B2B ассистент для первичного приема заявок по зерновым сделкам и операционный бэк-офис для менеджеров.

Цель MVP:

- получить сообщение клиента;
- извлечь из него структурированные параметры сделки;
- довести диалог до квалифицированной заявки;
- передать заявку менеджеру в CRM-контуре.

## 2) Кто пользуется системой

- Клиент (веб-чат): отправляет свободный текст.
- Ассистент: ведет диалог по воронке квалификации.
- Менеджер: работает в админке с лидами, статусами и каталогами.
- Администратор: управляет пользователями, справочниками, настройками.

## 3) Что входит в функционал

- Чат и потоковый ответ (`/api/chat/stream`).
- Детерминированное извлечение фактов (товар, объем, регион, контакт и т.д.).
- CRM-модель лидов и задач.
- Справочники: культура, регион, транспорт, базис поставки.
- Каталоги: шаблоны качества, ценовые политики, лоты.
- База знаний для RAG-ответов (`knowledge_article`).
- Админ UI: Dashboard, реестр, workspace лида, справочники, настройки.

## 4) Что не входит (ограничения MVP)

- Нет полноценного workflow-движка и BPM.
- Нет миграций БД (alembic отсутствует).
- Нет гранулярной RBAC по ролям в API.
- Нет автодозвона/интеграции с телефонией и 1C.
- Нет точного ценообразования в рантайме (только гипотеза оффера).

## 6) Основные бизнес-сценарии

- `purchase_from_supplier` — закупка у поставщика.
- `sale_to_buyer` — продажа покупателю.
- `logistics_request` — логистический запрос.
- `storage_request` — хранение/перевалка.
- `export_request` — экспортный запрос.
- `general_company_request` — общий/информационный запрос.

## 7) Жизненный цикл лида (в текущей реализации)

- `draft` — черновик (базовые данные не собраны).
- `partially_qualified` — есть минимальный контур + контакт.
- `qualified` — обязательные поля собраны.
- `handed_to_manager` — выставляется менеджером вручную из админки.
- `closed` — закрыт менеджером.
- `blocked` — блокирован (например, из-за guardrails).

## 8) Где лежит “истина” в системе

- По клиентскому диалогу: таблицы `chat_*`.
- По сделке/заявке: таблицы `crm_*`.
- По справочным значениям: `ref_*`, `catalog_*`.
- По знанийной базе: `knowledge_article`.
